## 1. 背景：QDS 表的大小问题

### 1.1 传统 QDS 表为什么这么大

SRT 除法的商位选择（QDS）传统上通过 **P-D 图**（Partial remainder vs Divisor diagram）构造。

P-D 图的基本原理：对于归一化除数 D ∈ [1,2) 和部分余数 PR ∈ (-2,2)，
每个可能的商位 m ∈ {-2,-1,0,+1,+2} 在 P-D 图上对应一个"选择区域"。
QDS 表的每一项就是：给定一个（pr_est 的截断值，d_hat 的截断值）→ 应该选哪个 m。

**传统 QDS 表的大小公式**：

```
传统大小 = (2^{δ+1}) × (2^{α+1})
```

其中：
- δ = d_hat 的 bit 数（不含整数位）
- α = pr_est 的 bit 数（不含整数位）

对于 SRT-4（ρ=2/3）：
- d_hat 取高 3 bit（δ=2，加 1 位整数 = 3 bit）
- pr_est 取高 7 bit（α=6，加 1 位符号 = 7 bit）
- 表大小 = 2^{2+1} × 2^{6+1} = **8 × 128 = 1024 项**

对于 SRT-8（ρ=6/7）：
- d_hat 取高 4 bit（δ=3，加整数位 = 4 bit）
- pr_est 取高 8 bit（α=7，加符号 = 8 bit）
- 表大小 = 2^{3+1} × 2^{7+1} = **16 × 256 = 4096 项**

### 1.2 核心观察：不需要所有项

**论文的关键洞察**：P-D 图上每个 m 的选择区域是连续的。我们不需要枚举每个（pr_est, d_hat）点来判断该选哪个 m，只需要知道**每个 m 的边界线（staircase line）位置**。这条边界线可以用截断后的 pr_est 和 d_hat 来描述。

> 打个比方：不需要查"北京所有门牌号 → 属于哪个区"的表，只需要知道每个区的边界在哪条街上。
> 边界需要的精度（几 bit）远低于门牌号全精度。

## 2. 第 3 章：QDS 表的理论基础

### 2.1 P-D 图回顾

P-D 图是 SRT 算法分析的核心工具：

- 横轴：除数 D（归一化后 ∈ [1,2)）
- 纵轴：部分余数 PR（归一化后 ∈ [-2,2)）
- 每个商位 m 对应 P-D 图上的一块区域，区域边界由两条曲线围成：
  - L(m)：PR ≥ m·D 的下界
  - U(m)：PR < (m+1)·D 的上界

### 2.2 论文的三个理论贡献（第 3 章）

#### （1）截断计算的代数推导

传统的 P-D 图方法用几何图形找边界（画矩形、找重叠区）。论文用**纯代数方法**推导截断后的边界公式，不依赖图形直觉，可以编程计算。

**核心公式（截断后的边界不等式）**：

```
对于 m > 0：
  m·D + ρ·D ≤ PR < (m+1)·D - ρ·D

对于 m < 0：
  (m-1)·D + ρ·D ≤ PR < m·D - ρ·D
```

其中 ρ 是冗余因子。

截断后，D 用高 δ bit 表示（记为 Z），PR 用高 α bit 表示（记为 P），不等式变为：

```
m·(Z - 2^{-δ-1}) + ρ·(Z + 2^{-δ-1}) ≤ P + 2^{-α-1}
```

论文证明了：**只要 α 和 δ 选得够，截断后的不等关系与原不等关系严格等价**——不会出现截断导致选错商位的反例。

#### （2）适用于任意冗余度的 QDS 构造方法

论文给出了可以**程序化自动生成** QDS 表的算法：

```
输入：基数 r，冗余度 ρ，商位集 {-a,...,+a}
输出：QDS 表（Ztrunc, Ptrunc 边界数组）

对每个商位 m ∈ 商位集:
  计算 m 在 P-D 图上的上界 U(m) 和下界 L(m)
  将 L(m) 和 U(m) 用截断 Z(δ bit) 和 P(α bit) 表示
  找出在截断精度内能区分相邻 m 的最小 δ 和 α
  记录每个 m 区间的边界点（Zflag, Pflag）
```

#### （3）任意冗余度下的商选择函数

论文给出了统一的商选择函数形式：

```
q = m,  当且仅当  L(m) ≤ PR < U(m)
```

其中 L(m) 和 U(m) 由截断后的不等式决定。

### 2.3 第 3 章的核心价值

第 3 章不是直接缩减表大小，而是**提供了可以自动生成正确 QDS 表的数学工具**。这为后续的缩减方法奠定了基础。

## 3. 第 4 章：截断迭代缩减方法

### 3.1 方法本质

第 4 章的核心思想：**不同商位 m 对 PR 和 D 的精度要求不同，没必要全表统一用最大精度。**

举例：
- 当 m=0（PR 很小时），不需要很高的截断精度就能判断"PR 接近零"
- 当 m=±2（PR 很大或很小时），边界附近才需要高精度区分"该选 ±2 还是 ±1"

### 3.2 迭代缩减算法

算法流程（论文 Algorithm 4.1）：

```
输入：基数 r，冗余度 ρ，商位集
输出：每个 m 区间的 Ztrunc(m) 和 Ptrunc(m)

对每个商位 m:
  初始化 Ztrunc = 1 (最少 1 bit d_hat)
  初始化 Ptrunc = 1 (最少 1 bit pr_est)
  
  迭代:
    用当前 (Ztrunc, Ptrunc) 构造截断后的边界
    检查：边界之间是否有足够间距区分相邻商位？
    如果间距不够:
      增大 Ztrunc 或 Ptrunc（优先增大贡献更大的那个）
      继续迭代
    如果间距足够:
      记录 (Ztrunc, Ptrunc)
      结束
```

**关键决策**：每次迭代优先增大 Z 还是 P？论文给出了判断准则——看当前哪个维度的截断误差对边界判断影响更大。通常 Ptrunc 需要更多 bit（因为 PR 的尺度变化范围更大）。

### 3.3 结果

**表 4.1：SRT-4 ρ=2/3 的截断结果**

对每个商位 m，迭代计算得到的截断位数：

| m | Zflag (d_hat 的高位特征) | Ztrunc | Pflag (pr_est 的高位特征) | Ptrunc |
|---|--------------------------|--------|---------------------------|--------|
| 2 | 1.001~1.111 (二进制) | 3 | 0.01101~0.1011 | 5 |
| 1 | 1.001~1.111 | 3 | 0.00101~0.10011 | 5 |
| 0 | — | — | — | — |
| -1 | 1.001~1.111 | 3 | -0.01101~-0.10011 | 5 |
| -2 | 1.001~1.111 | 3 | -0.01101~-0.1011 | 5 |

**结果解读**：
- Ztrunc = 3 表示 d_hat 只需要高 3 bit（即我们当前的 3 bit d_hat 就够了）
- Ptrunc = 5 表示 pr_est 只需要高 5 bit（比我们当前的 7 bit 少 2 bit！）
- 但不同 m 的 Pflag 起始位置不同，所以实际实现的 QDS 表并不是统一的 3×5 网格

**表大小**（论文 4.5.2 节）：
- 论文方法得到的 SRT-4 ρ=2/3 QDS 表：**546 项**
- Kornerup [24] 方法：523 项
- 传统方法：1024 项
- 缩减比例：**~47%**

**为什么不是简单的 Ztrunc×Ptrunc 网格？**

因为截断后的边界不是矩形，而是"阶梯线"（staircase line）。每个 m 的边界在不同 Z 值下需要的 Ptrunc 不同。表 4.2 展示了实际计算出的每个 Ztrunc 值对应的 Ptrunc：

```
Z=1.001 → Ptrunc=5
Z=1.010 → Ptrunc=5
Z=1.011 → Ptrunc=4
Z=1.10  → Ptrunc=5
Z=1.11  → Ptrunc=5
Z=10.00 → Ptrunc=4
Z=1.1   → Ptrunc=5
Z=10.0  → Ptrunc=5
```

Ztrunc=3（5 种 Z 值） + Ztrunc=2（2 种 Z 值） + Ztrunc=1（1 种 Z 值）= **不是规则网格，而是 8 个 (Z, P) 边界点**。

### 3.4 表 4.3：SRT-8 ρ=6/7 的结果

论文 4.5.3 节同样给出了 SRT-8 ρ=6/7 的结果：

| m | Ztrunc(典型) | Ptrunc(典型) |
|---|-------------|-------------|
| 6 | 4 | 5 |
| 5 | 4 | 5 |
| 4 | 3 | 4 |
| 3 | 3 | 4 |
| 2 | 2 | 4 |
| 1 | 2 | 5 |
| -1~-6 | (对称) | (对称) |

- 迭代缩减后的 QDS 表：**336 项**
- Kornerup [24]：808 项
- 传统方法：1024~4096 项（取决于实现细节）

### 3.5 第 4 章小结

| SRT | ρ | 传统大小 | 迭代缩减 | 缩减比例 |
|-----|---|---------|---------|---------|
| SRT-4 | 2/3 | 1024 | **546** | **47%↓** |
| SRT-8 | 6/7 | 1024~4096 | **336** | **67~92%↓** |

## 4. 第 5 章：最小截断理论

### 4.1 从"迭代缩减"到"最小截断"

第 4 章的迭代缩减找到了"足够"的截断精度，但不一定是"最小"的。

第 5 章在迭代缩减的基础上，做了一步优化：

> **对每个 m 的边界，精确找出它需要的 Ztrunc 和 Ptrunc 的最小值。**
> 方法：在迭代缩减的结果上，逐 bit 缩小截断，检查边界间距是否依然足够。

### 4.2 对除数截断的优化调整

论文 5.2 节提出，d_hat 的截断边界可以**略微调整**（上下浮动半个 LSB），以进一步减少 Ptrunc 需求：

```
调整前：Z(δ) = 截断到 δ bit 的 D
调整后：Z'(δ) = 截断到 δ bit 的 D，并在边界上做 ±2^{-δ-1} 的调整
```

这种调整改变的是 **d_hat 的量化方式**，不改变硬件实现（本来也是截断取高 bit）。

### 4.3 最小 PR 截断计算

论文 5.3 节给出了最小截断计算算法：

```
对每个商位 m，在其边界区域内:
  1. 用当前 (Ztrunc, Ptrunc) 检查边界间距
  2. 如果间距 > 最小可区分间距（1个LSB）:
     - 尝试减小 Ptrunc（即 pr_est 少用 1 bit）
     - 检查减小后是否仍能区分相邻 m
     - 如果能：采用更小的 Ptrunc
     - 如果不能：用之前的 Ptrunc
  3. 对 Ztrunc 做同样的优化尝试
```

### 4.4 最小截断后的结果

**表 5.3（SRT-8 ρ=6/7）**：

| m | Zflag 范围 | Ztrunc_min | Pflag 范围 | Ptrunc_min |
|---|------------|-----------|-----------|-----------|
| 6 | 1.0000~1.1111 | 4 | 0.71875~1.3125 | 5 |
| 5 | 1.000~1.111 | 3 | 0.59375~1.125 | 5 |
| 4 | 1.0~1.75 | 3 | 0.46875~0.84375 | 4 |
| 3 | 1.0~1.75 | 2 | 0.34375~0.625 | 4 |
| 2 | 1.0~1.5 | 1 | 0.21875~0.34375 | 4 |
| 1 | 1.0 | 0 | 0.0625 | 4 |
| -1~-6 | （对称于正半轴） |

**关键发现**：m=±1 只需要 Ztrunc=0（即不依赖 d_hat）！因为 PR 足够小时，不管除数多大，q=±1 一定是安全的。

### 4.5 最小截断 QDS 表大小

| SRT | ρ | 迭代缩减 | 最小截断 | 再缩减比例 |
|-----|---|---------|---------|-----------|
| SRT-4 | 2/3 | 546 | **~314** | **42.5%↓** |
| SRT-8 | 6/7 | 336 | **232** | **31%↓** |

论文第 5 章的标题数据：

> SRT-8 ρ=6/7：最小截断后 **232 项**，对比传统全表缩减 **71.3%**，
> 在 Kornerup [24] 的 336 项基础上再缩减 **31%**。
>
> SRT-4 ρ=2/3：最小截断后 **~314 项**（论文 5.4.3 节），
> 在 Kornerup [24] 的 546 项基础上再缩减 **28.6%**。

### 4.6 为什么表大小不等于 Ztrunc×Ptrunc

由表 5.3 可见，Ztrunc 和 Ptrunc 并不是全表统一的数值。**每个 m 有自己独立的 (Ztrunc, Ptrunc) 对**：

- m=6 需要 (4, 5)→ 但只有特定 Z 范围需要 5 bit，其他只需要 4
- m=1 只需要 (0, 4)→ 不依赖 d_hat

所以表的实现不是 16×256 的 ROM，而是：

```
if pr_est >= threshold[6][Z]:
    q = 6
elif pr_est >= threshold[5][Z]:
    q = 5
...
```

每个 threshold[m][Z] 是一个预计算的常数，由 Pflag 和 Ptrunc 决定。

## 5. 第 6 章：SRT-8 实现

### 5.1 架构

论文第 6 章基于最小截断 QDS 表实现了 SRT-8（ρ=6/7）除法和开平方共用模块。

**商位集**：{-7,-6,...,+6,+7}（最大冗余度，ρ=6/7）

**QDS 表**：232 项（表 6.2）

**结构特点**：
- **P_Path**：PR 更新路径，使用 3:2 CSA
- **Q_Path**：商位累积路径，使用 on-the-fly conversion
- **两级选择优化**（论文 6.3.2 节）：将 232 项的 QDS 表分成 4 个比较层，减少每层的比较器数量

### 5.2 两级选择优化

这是论文 6.3.2 节的关键硬件优化技术。232 项的 QDS 表如果直接实现为 8 级比较器（m=±1 到 ±7），每级的比较器宽度都不一样，电路面积和延迟都大。

**两级选择法**：

```
第一级：粗选择
  将 232 个表项按 pr_est 范围分组
  每组对应 2~3 个候选 m 值
  用 pr_est 的高位快速确定属于哪组（约 100 个比较器）
  
第二级：细选择
  在粗选择锁定的组内，用 pr_est 的低位精确确定 m 值
  每组内只有 2~3 个候选，比较器面积极小
```

**效果**（论文表 6.3）：
- 粗选择平均搜索长度：10.35（232 个候选中先缩到 ~10 个）
- 细选择平均搜索长度：4（10 个中最后精确到 1 个）
- 总平均比较次数：**14.35**（vs 直接比较 232 次的传统做法）
- 注意这里 ASL 的单位是"候选个数"，不是门延迟。实际硬件中比较器是并行的，延迟只取决于比较树深度。

### 5.3 综合结果

论文使用 **PyRTL** 生成 RTL，综合到 **65nm CMOS**：

| 指标 | 传统 QDS 表 | 最小截断 QDS 表 | 改善 |
|------|-----------|---------------|------|
| 单元面积 | 228,591 µm² | **154,404 µm²** | **32.5%↓** |
| 组合逻辑 | 200,079 | 125,892 | **37%↓** |
| 时序延迟 | 114,966 ps | **57,167 ps** | **50.3%↓** |
| 动态功耗 | 12.22 mW | **10.90 mW** | **10.8%↓** |

**注意**：这里的 "时序延迟" 是论文使用的 65nm 工艺下的值。论文 6.4.2 节提到这个延迟来自 PyRTL 的时序估计（不是综合后的 STA），所以 114 ps 和 57 ps 是设计内部的相对比较值，不是按比例缩小到 7nm 之后的绝对延迟。

### 5.4 论文第 6 章的关键启示

1. **QDS 表可以不再是 ROM**：232 项的 QDS 表实现为比较器树，而不是 ROM 查找
2. **两级选择优化**：用 pr_est 高位做粗分类，低位做精确判定，大幅减少比较器
3. **面积和时序双赢**：QDS 表变小同时改善了延迟（组合逻辑路径变短）
4. **同样适用于 SRT-4**：我们的 1024 项表按同样方法可缩到 ~314 项

## 6. 对我们项目的意义

### 6.1 需要实现的 QDS 表

我们的多模架构需要 3 套 QDS：

| QDS | 传统大小 | 论文方法预测 | 我们的实现方式 |
|-----|---------|------------|--------------|
| SRT-2（ρ=1/2） | 64 | ~20 | 2~3 级比较 |
| **SRT-4（ρ=2/3）** | **1024** | **~314** | **4~5 级比较** |
| **SRT-8（ρ=4/7 或 6/7）** | **1024~4096** | **~232** | **4~5 级比较** |

### 6.2 实现策略

论文方法可以直接应用到我们的设计：

**第 1 步**：对 SRT-2（ρ=1/2）运行迭代缩减算法，求出最小截断

**第 2 步**：对 SRT-4（ρ=2/3）运行迭代缩减算法

- 当前我们用的 pr_est=7 bit，d_hat=3 bit
- 论文建议：Ptrunc=5~6 bit 就够（少 1~2 bit）
- 预期表大小：**314 项**（而非 1024）

**第 3 步**：对 SRT-8（ρ=4/7 或 6/7）运行迭代缩减算法

- pr_est 取 8 bit（或按论文结果只需 5~6 bit）
- d_hat 取 4 bit（或按论文结果只需 3~4 bit）
- 预期表大小：**232 项**（ρ=6/7）或更小（ρ=4/7 时可能略大）

### 6.3 硬件实现形式

论文方法得到的 QDS 表**不是 ROM**，而是**一组比较器树**：

```
// 传统：ROM 查表
q = qds_rom[pr_est][d_hat];    // 1024 项 ROM，慢

// 论文：分段比较
if      pr_est >= T[m_max][z] : q = m_max;
else if pr_est >= T[m_max-1][z] : q = m_max-1;
...
else                             : q = -m_max;
```

每个 T[m][z] 是预计算好的阈值常数（由每个 m 的 Pflag 和 Ptrunc 决定）。

### 6.4 与推测架构的结合

在多模推测架构中，5 路推测需要**5 个并行的第二级 QDS**。

如果每路 QDS 用最小截断方式实现（~232 比较器），5 路总共约 ~1160 比较器。
如果每路 QDS 用传统全表（~1024 项），5 路需要 **5120 比较器**。

**差 4.4 倍**——这就是我们要用论文方法的原因。

## 附录：与我们的设计的关系

### 我们当前用的 SRT-4 表

目前 pkg 里的 `qds_lookup` 用 7 bit pr_est + 3 bit d_hat → 1024 项有效组合。
论文建议：pr_est 只需要 5~6 bit → 可缩至 ~314 项。
阈值不会变（M2=12/14/16/16/18/20/22/24，M1=4/4/4/4/6/6/8/8），变的只是**比较器的实现形式**。

### 对推测架构的影响

推测架构的 5 路中，每路需要第二级 QDS：
- SRT-16 模式：第二级是 SRT-4 QDS → ~314 比较器
- SRT-32 模式：第二级是 SRT-8 QDS → ~232 比较器

两种模式的第二级 QDS 都是论文方法实现的小型比较器树，不是 ROM。

### ebox 表的定位

`reference/srt4_ebox_qds.txt` 中的表来自某流片芯片的 SRT-16 模块。它实际上是一个**未经缩减处理的 QDS 表**（1024 项）。如果我们用论文方法重做，预计 ~314 项就可以，硬件代价只有 ebox 方案的 1/3。
